Changes that has been made

Fonts

The fonts we had chosen initially brought up some compatibility concern and we opted to change the fonts as well as include fallback fonts. For the header as well as the navbar fonts we had initially chosen Tajawal but ended up using Trebuchet MS. For the paragraphs we had initially planned to use lora. Where lora is supported it will be used. but in most cases Times New Roman will be the paragraph font.

Width of body

The body width was originally specified to be 900px. But as we found some pages to be a little cramped with only 900px width we made the decision, with readability and ease of use in mind, to make use of the screen space and set the width to 75% of screen width. With a width of 75% we achieved a more spacious and clearer webpage.

Nav bar

A lot of changes were made to the navbar. Not to the original design but how it functions. Because of the relatively few pages you will have to access we felt it would be easier to keep the design as is on mobile pages as well and not make a dropdown but instead show all the different pages you can access in the navbar right away.

Footer

The footer contains more info than was displayed in the mockup. We also wanted to display the sponsors in the footer as well as we imagine that's what an actual sponsor for a small football club would have wanted. We also looked at other small websites to see they had sponsors in their footer. More contact info was also put in the footer and links to various social media sites related to NTNU as there were none directly related to Webtek.

Index page

Changes to calendar.html

We changed the position of the paragraph to be above the calendar list instead of under.

Calendar and results lists

We did not specify how we were going to display this on the page in P2 but the initial idea was to just write it out in plain HTML. As we did this we saw how confusing and untidy it was to both edit and list new results. And for someone to maintain and update the list it would take a lot longer to do this in HTML than with a tidy and clean list in Javascript. So instead of focusing on a dropdown for the nav we prioritized structuring and displaying results and upcoming matches in Javascript, as the navbar as previously mentioned was fine for the end user on mobile we believe, while maintaining the lists were difficult and cumbersome.
