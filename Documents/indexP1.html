Administrative Details

We choose an imaginary client. Our client is a small football team that wants more members. The team is called Webtek FC and is located in Trondheim.

Purpose and goals

The main purpose with the site is to spread awareness about the football club to the people in Trondheim, since it is where the club is located. The club is fairly new and needs a website such that people can learn about it. It is important to make sure that the reader knows that it is meant for both kids, teenagers and adults. This will be done by using the bold element such that it will be easy to see it. It is free to join Webtek FC, hence one of the club’s wishes to have more coaches that can help. We want to make a separate page for this such that the coaches that wish to help can read more about it there. To attract more players the club wants to have a spinning wheel on the website so that visitors of the site have a chance to win prizes, such as water bottles, shoes, t-shirts etc. And lastly the website needs a page where you can read what players who have already been there have said about the club, together with FAQ. Thus our overall goal will be to create such a website where all these demands will be met, and where the average user can easily navigate through the website.

Audience

The website’s main audience are those who are interested in Webtek FC. It may be a parent who needs information about when her son is training, it may be a person who is interested in joining the football club, or it may be a person who would like to learn about the history of the club. It may even be used by companies who would like to sponsor the club. The site’s audience include parents, active members of the club, sponsors, people interested in joining the club and fans.

The content of the site and how it is organized

About

This section will include information about the football club’s history. This includes how and when the club was formed, notable players and matches, and information about the current team. This section will be interesting and insightful, but also easy to read, in order to keep the user interested and entertained.

News

This section will provide news about the team, like for example interviews with new players and information about upcoming matches or tournaments. This section will consist of articles. It will help the user keep up with updates about the team.

Join the club

This section will contain information about how to join the club, whether it be a child being signed up for the children’s team or a parent signing up to volunteer. This will be done by filling out a form provided in the section.

Upcoming matches and tournaments

This section will consist of different subsections:

  • Ticket prices for the current season
  • Information about season pass
  • Directions to the stadium / information about parking

Section a will provide information about the different ticket prices. For example, discount for students, seniors and children, or VIP prices. In section b the user will be able to get information about what is included in the season pass, like for example a free drink with each match and better seats. It will also inform the user where they can buy the season pass. Section c will show the user how to get to the stadium by car or public transport. It will also contain information about parking. These subsections will help the user plan their visit to the stadium in the possible way, and therefore it needs to present the information in a simple way. Pictures and/or a map are included.

Results

This section will include the most recent results from matches. Each match result will also contain the opposing team, where the match was played, and the time and date for the match. In this section it will be possible for the user to get a quick overview of the results.

Footer

The footer will contain the club’s e-mail address and phone number. It will also contain links to their social media, for example Facebook and Instagram. It will also be possible to sign up for the newsletter here. By gathering the contact information in the footer, the user can access the information with ease from any section in the web page.

Functional and Non-Functional Requirements

Functional requirements: Any requirement which specifies what the system should do.

  • The system must allow users to identify where certain information is. (Where the club is located, how to become a member, view upcoming games)
  • The system must allow users to contact personnel if they encounter a problem. (support page/contact page).
  • The system must allow users to ask for information about the club and what sort of volunteer work they can participate in.

Non-Functional requirements: Any requirement which specifies how the system performs a certain function.

  • Every webpage must look clean and not too complex
  • The website should not overwhelm a user with too much information on one webpage.
  • The system must be accessible on smaller devices such as mobiles and tablets. (responsive web design)
  • A web page should render within 2 seconds. (https://developers.google.com/speed/pagespeed/insights/)
  • The website should be portable with most web browsers.
  • The design should be pleasant to use and easy to learn.
